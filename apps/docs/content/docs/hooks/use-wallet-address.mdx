# useWalletAddress

The `useWalletAddress` hook provides access to the current wallet's address in both string and parsed formats, along with connection status information.

## Usage

### Direct Import
```tsx
import { useWalletAddress } from '@connectorkit/hooks'

function WalletInfo() {
  const { address, addressParsed, connected, connecting } = useWalletAddress()
  
  if (connecting) return <div>Connecting...</div>
  if (!connected) return <div>Please connect your wallet</div>
  
  return (
    <div>
      <p>Address: {address}</p>
      <p>Short: {address?.slice(0, 4)}...{address?.slice(-4)}</p>
    </div>
  )
}
```

### Copy Hook Code
```tsx
'use client'

import { useMemo } from 'react'
import { useArcClient } from '../core/arc-client-provider'
import { address as parseAddress, type Address } from '@solana/kit'

export interface UseWalletAddressReturn {
  address: string | null
  addressParsed: Address | null
  connected: boolean
  connecting: boolean
}

export function useWalletAddress(): UseWalletAddressReturn {
  const { wallet } = useArcClient()

  const addressString = wallet.address ?? null
  const addressParsed = useMemo(() => {
    if (!addressString) return null
    try {
      return parseAddress(addressString)
    } catch {
      return null
    }
  }, [addressString])

  return {
    address: addressString,
    addressParsed,
    connected: wallet.connected,
    connecting: wallet.connecting,
  }
}
```

## Parameters

This hook takes no parameters.

## Return Value

- `address: string | null` - The wallet address as a string, or null if not connected
- `addressParsed: Address | null` - The wallet address parsed using @solana/kit, or null if not connected
- `connected: boolean` - Whether the wallet is connected
- `connecting: boolean` - Whether the wallet is currently connecting

## Use Cases

- **Display wallet address**: Show the connected wallet's address to users
- **Address validation**: Use the parsed address for type-safe operations
- **Connection status**: Check if a wallet is connected before performing operations
- **Conditional rendering**: Show different UI based on wallet connection state
- **Address formatting**: Display shortened versions of the address for better UX

## Notes

- The hook automatically handles address parsing and error handling
- Returns null values when no wallet is connected
- The `addressParsed` field provides a type-safe Address object for use with @solana/kit functions
- Connection states are reactive and will update when the wallet connection changes
